## An Open-Source Blueprint for Community Self-Sufficiency
## Pattern: 3 → 7 → 13 → ∞

### What This Is

A complete, field-tested playbook for building a community sovereignty hub from zero. One person, one truck, one rented space — growing into a 13-department community institution that provides water, food, shelter, energy, medicine, communication, manufacturing, security, transportation, trade, governance, knowledge, and culture.

Every pillar follows the same 7-layer model:

```
Layer 0: SURVIVAL     — You have nothing. Improvise with $0.
Layer 1: PREPAREDNESS — One product under $100 that changes everything.
Layer 2: STOCKPILE    — Raw materials that appreciate 3-30x in crisis.
Layer 3: PRODUCTION   — Make things. Sell them. 50% margins.
Layer 4: COMMERCE     — Curate the best. Charge for your knowledge.
Layer 5: TEACHING     — Classes, guides, installs. Highest-margin revenue.
Layer 6: INNOVATION   — Technologies that change the game entirely.
```

Every pillar follows the same 3 phases:

```
SURVIVE → BUILD → SCALE
```

The math: **13 pillars × 7 layers × 3 phases = 273 action points.** Each one answers: *"I am in [PHASE], working on [PILLAR], at [LAYER]. What do I do next?"*

And each pillar contains all 13 within it (the fractal): **13 × 13 × 7 × 3 = 3,549 coordinates.**

### How to Start

**You have $0:**
Read the Layer 0 section of any pillar. Every one starts with what you already have.

**You have $50:**
Buy a Sawyer Squeeze water filter ($45). You now have infinite clean water from any creek. That's Pillar 1, Layer 1, done.

**You have $500:**
Build 10 gravity filter kits ($30 each) and sell them for $99 each. You just made $690 and activated Water, Manufacturing, and Trade simultaneously.

**You have $5,000:**
Full hub setup touching all 13 pillars. See `SYSTEM_ARCHITECTURE.md` for the complete boot sequence.

### How to Contribute

Every improvement you make to your local implementation can be submitted back to this blueprint so every other hub benefits.

- **Fix a supplier link** — prices change, sources disappear. Update them.
- **Add a local variation** — what works differently in your climate/region?
- **Share a revenue number** — real data makes the projections better.
- **Document what didn't work** — failures are as valuable as successes.
- **Design a new product** — if you invented a better filter housing, share the STL.

See `CONTRIBUTING.md` for how to submit improvements.
